Wider environment

As PLU_ARCHIVER functions as a tiny (perhaps not that much) cog in the complicated ISO 17100 workflows at REDDO Translations, this section is to give you an idea where this all fits in with the bigger picture.

In practical terms, from this section you will get a glimpse of where PLU_ARCHIVER comes into play and what it can do ...perhaps also what it can't do. The purpose of this document is not to discuss the workflows in detail - if you have any doubts, refer to the REDDOnomicon.

Process overview

 

There are two tasks in the general process flow that are important for resource management and in the bigger picture contribute to where PLU_ARCHIVER is used.

Project management (PM)

 

 

 

As part of the project management job, the PM uses a MemoQ template to create (among others) two heavy resources:

A project TM and a project TB.

 

 

 

TM/TB export (TEX)

 

 

As part of the TM/TB export job, the memoQ server API communicates with the Plunet server API to export heavy resources.

*As at April 2023: since the MemoQ API does not include a handle that would identify the Working TM and Target TB, this API call exports ALL resources attached to a project.

This is one of the main reasons why PLU_ARCHIVER was created - as sometimes the export creates 500 TM and 500 TB exports, the server storage was flooded.

 

 

 

Downstream process

TEX jobs are executed in the business-as-usual context on a daily basis, generally without the need for user intervention.

In the background, there is a separate workflow that requires human input, focused on managing resources from the technical and linguistic point of view.

This workflow requires the involvement of the PM again, the senior production employees (Head of Quality, Head of Review) who handle more complex tasks, regular production employees (translators) who perform jobs requiring linguistic skills and translation proficiency.

However all of them would not be able to work without input material, which is created through the use of PLU_ARCHIVER:

Resource clean (RCL)

 

As part of the RCL job, the designated operator runs PLU_ARCHIVER to achieve the following:

  • archive orders in Plunet (files are moved from the server, the order details remain available)

  • free up Plunet server storage by moving archived files to the alternative storage location

  • maintain resources in MemoQ (delete project resources, populate global resources, create new resources if necessary)

  • archive and backup Plunet and MemoQ content in the designated backup location
