Modules and components

This section explains the individual modules and components used in the PLU_ARCHIVER. This is a general overview to give you an idea where to look for the functionality you might be interested in - though this will mostly be relevant for the Developer user.

Therefore, some obvious coding-related issues are not discussed at all and should be clear to the Developer user - e.g. "__init__.py" files are not discussed for obvious reasons.

If there is any functionality/info that could be relevant and/or interesting to the Operator user, it is marked in RED.

Elements marked in BLUE constitute potentially useful separate tools that may be deployed in other scenarios related to MemoQ/Plunet/REDDO workflows, beyond the PLU_ARCHIVER.

Elements marked in GREEN are core components for PLU_ARCHIVER, usually aggregating different tools into the desired workflow.

Elements marked in BROWN are deprecated old tools that have either been abandoned or re-used in other components; they are retained here for audit trail purposes and... lack of time!

 

 

The individual components seen in the version 1.0 PLU_ARCHIVER Git repo are:

common

CatalogFunctions.py - file system functions used for packing files and sorting archive files,

try_compress_directory - function used for packing single order folders into .ZIP archives, including a try section

multi_packer - function aggregating the try_compress_directory function, packaging an entire input list and providing feedback to the user

sorter - function for sorting packed order .ZIP archives into the target year\month structure

Csv.py - for reading CSV encoding and delimiter

determine_csv_properties - reads and returns encoding and delimiter as a named tuple

LanguageTools.py - for handling language code variants in MemoQ, language code conversion between formats and mapping TMX/TM-TB name language codes into the MemoQ standard

LanguageDescription - class that stores the coding parameters and mappings for a language

LanguageDescriptionCollection - aggregates LanguageDescription instances

get_[...] - functions for converting various language code formats

LoggingTools.pl - creates log file
notifications

Email.py - infrastructure for e-mailing log file as appendix and log messages in body; includes log descriptors used in the entire system
plu_archiver

PLU_ARCH.py - main body of the system, agrregating all functionalities as per specifiction

searcher - compiles a list of order folders for further processing

deleter - deletes unnecessary contents of the TEX job folder, leaving only project resources; checks for various error conditions to feed into special cases

legacy_archive_searcher - looks for archives from previous runs that have not been deleted/moved properly

archive_volume_summarizer - determines the volume of archives created to confirm adequate storage in target location

space_checker - checks target storage location capacity to prevent loss of files upon copying

archive_mover - moves packed order archives to the proper storage site

plu_archiver - main function, aggregating sub-functions into main process flow

- create candidate list

- delete waste files

- pack for archiving

- move archive ZIP files

RESOURCE_PROCESSOR.py - main file for the system, containing the entire main workflow, aggregating all sub-routines

1) validates CSV for proper contents

2) parses CSV to create candidates for further processing

3) separates multilingual projects for manual processing

4) runs the PLU_ARCHIVER (deletes waste content, separates project resources to the sandbox, packs orders into ZIP archives, creates a folder structure and moves content to archive location)

5) runs the RES_PROCESSOR (creates pooled TMs and TBs from sandbox contents + client TMs/TBs, separates orders without resources, imports into global resources, deletes project resources, creates pooled TBs for review, sends out log via e-mail)

TermBasesTools.py - tool for working with TB merge operations when running the RES_PROCESSOR and creating new TB import files (CSV) with the proper structure
Seclore_TPT Seclore_parser.py - tool file for the Seclore parsing app, deprecated and obsolete due to insignificant volume of Seclore jobs, abandonware

CONFIG.py - config file for the entire system, containing all variables that are configurable per deployment; CONTAINS SENSITIVE/CONFIDENTIAL data, including access credentials, DO NOT ADD TO GIT

CONFIG_TEMPLATE.py - template config file for the entire system, containing all variables that are configurable per deployment; added to GIT and must be renamed to CONFIG.py on the local instance

Seclore.py - main file for the Seclore parsing app, deprecated and obsolete due to insignificant volume of Seclore jobs, abandonware

year+month_file_crawler.py - ad-hoc tool for splitting a set of orders into year\month structure; if need be, may be separated into a separate .exe
