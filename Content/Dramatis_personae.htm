Dramatis personae

This section explains the background of the two types of recipients of this documentation: dev user and operator.

Below you will find some information on the skills, competences, knowledge and tasks of the two groups of individuals that might be interested in using RES_PROCESSOR.

Dev User

The RES_PROCESSOR was written between 2021 and 2023 by internal REDDO resources with the support of an external experienced developer.

Once the version 1.0 has been released, the initial team moved on to other projects, as all the initially specified functionalities have been delivered.

We have done our absolute best to remove all the bugs that we could find during testing.

However, we are sure that as the business grows, there will be additional requirements that the software will need to support.

There will of course be ongoing development in the underlying software, MemoQ and Plunet, which might necessitate maintenance.

So if you are a new developer who is to maintain or upgrade the functionalities of RES_PROCESSOR, this is the place to start.

This documentation details the components of the software available in the relevant Azure repo, describes the tasks and functions.

If you're entirely new to the concept behind RES_PROCESSOR, check out the Wider environment, What is this and What in tarnation.

If you have an idea of what workflows at REDDO are serviced by RES_PROCESSOR, perhaps start with just reading the code after you checkout the RES_PROCESSOR section, which details the software components.

 

 

Operator User

 

 

If you're here, you've probably been charged with resource maintenance at REDDO Translations or are trying to use one of the components of RES_PROCESSOR to facilitate your work.

Using RES_PROCESSOR currently requires some IT skills, as we have not developed a GUI for the software and it has to be run from the command line.

Be not afraid, in the end it's all quite simple and there are few steps you need to follow in the software itself.

Before you start, you should have a fairly good understanding of the resource workflows at REDDO Translations, so the Wider environment section is a good place to start.

Please remember that your role is that of facilitation and not doing the work yourself - you have the support of Production, Project Management, Reviewers and IT.

You will receive input from the PM in charge of resource maintenance, who is also able to provide instructions.

Go take a look around the different sections, it's really quite simple in the end.
